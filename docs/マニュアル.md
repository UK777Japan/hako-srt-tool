## 1. ツールの概要

### ハコ割り生成ツールとは

「ハコ割り」とは、動画字幕の原稿（セリフを読みやすい単位で区切ったテキスト）のことです。  
このツールは、**音声・映像ファイル** と **ハコ割りテキスト** を組み合わせることで、  
各ハコのタイムスタンプを自動的に割り出し、**SRT 形式の字幕ファイル** を生成します。

### できること

- MP3, WAV, MP4, MPG, TS, M4A, AAC, FLAC, OGG に対応
- 上限 10GB（映像から音声を抽出するため、実際の処理は数百 MB 以内）
- Whisper AI による高精度な音声認識（99 言語対応）
- DP アライメントによるハコと音声の自動対応付け
- ローカル処理（無料・インターネット不要）と OpenAI API モードを選択可能
- 生成した SRT を Premiere Pro, DaVinci Resolve, Final Cut Pro などにそのままインポート可能

### 処理の流れ

```
音声・映像ファイル ──┐
                      ├─→ Whisper AI（音声認識）─→ DP アライメント ─→ SRT ファイル
ハコ割りテキスト  ──┘
```

## 2. システム要件

| 項目 | 最小要件 | 推奨 |
|------|---------|------|
| OS | Windows 10 / macOS 10.15 | Windows 11 / macOS 13+ |
| RAM | 8 GB | 16 GB 以上 |
| ストレージ | 5 GB 空き（Docker + モデル） | 10 GB 以上 |
| CPU | 4 コア | 8 コア以上 |
| インターネット | 初回のみ必要（モデルダウンロード） | — |
| Docker Desktop | 必須（無料） | 最新版 |

> **GPU について**: GPU がない場合も CPU で動作します。  
> ただし長い動画（30 分以上）は CPU だと数十分かかる場合があります。  
> その場合は後述の OpenAI API モードをご利用ください。

ブラウザで `http://localhost:8503` を開いた画面の説明です。

│  📄 ハコ割り生成ツール                    ⚙️ サイドバー │
│  音声ファイルとハコ割りテキストをアップロードして     │
│  SRT を生成します。                                  │
│                                                     │

│  │音声・映像ファイル│  │ハコ割りテキスト (.txt)  │   │

│  │   クリック    │  │                        │   │
│  └──────────────┘  └──────────────────────────┘   │
│                                                     │
│  ○ 空行区切り  ○ 改行区切り                         │
│                                                     │

│  │         ▶ SRT を生成                   │        │
│  └────────────────────────────────────────┘        │
└─────────────────────────────────────────────────────┘

### 操作手順

**ステップ 1: 音声・映像ファイルをアップロード**

- 「音声・映像ファイル」エリアをクリックしてファイルを選択
- またはファイルをドラッグ&ドロップ
- 対応形式: `MP3` `WAV` `MP4` `MPG` `MPEG` `TS` `M4A` `AAC` `FLAC` `OGG`

**ステップ 2: ハコ割りテキストをアップロード**

- 「ハコ割りテキスト (.txt)」エリアをクリックしてファイルを選択
- テキストファイルの文字コードは UTF-8 で保存してください

**ステップ 3: 区切り方式を選択**

| 選択肢 | テキストの形式 | 例 |
|--------|-------------|-----|
| 空行区切り | ブロック間に空行がある | 下記 A 参照 |
| 改行区切り | 1 行 = 1 ブロック | 下記 B 参照 |

**テキスト形式 A（空行区切り）:**
```
おはようございます。
本日は晴天なり。

今日の予定を
お伝えします。

午前 10 時から
会議が始まります。
```

**テキスト形式 B（改行区切り）:**
```
おはようございます。本日は晴天なり。
今日の予定をお伝えします。
午前 10 時から会議が始まります。
```

**ステップ 4: 「▶ SRT を生成」をクリック**

進捗がリアルタイムで表示されます:
```
SRT を生成中...
  ステップ 1/6: 音声変換中 (ffmpeg)
  ステップ 2/6: Whisper 音声認識中（しばらくお待ちください）
  ステップ 3/6: DP アライメント計算中
  ステップ 4/6: タイミング調整中
  ステップ 5/6: タイムスタンプ調整中
  ステップ 6/6: SRT 書き出し中
  完了！
```

**ステップ 5: ダウンロード**

完了後、ダウンロードボタンが表示されます:
- `⬇ output.srt をダウンロード` — 字幕ファイル
- `⬇ output.txt をダウンロード` — タイムスタンプ付きテキスト

ファイルはデスクトップの `ハコ割り生成ツール_output` フォルダにも保存されます。

## 6. 入力ファイルの準備

### 音声・映像ファイルのポイント

- **動画ファイル（MP4, TS, MPG）**: 音声トラックが自動的に抽出されます
- **音声品質**: クリアな音声ほど認識精度が上がります
- **BGM・効果音**: 音声に重なっている場合は認識精度が下がることがあります
- **ファイルサイズ**: 上限 10GB（映像 → 音声変換後は大幅に小さくなります）

### ハコ割りテキストのポイント

- **文字コード**: UTF-8（メモ帳で「名前を付けて保存」→「UTF-8」を選択）
- **内容の一致**: 音声の内容とテキストが一致しているほど精度が上がります
- **句読点・記号**: ある程度含まれていても問題ありません
- **余分な空白行**: 空行区切りモードでは、連続する空行は 1 つの区切りとして扱われます

## 7. 詳細設定

サイドバー（画面左端の「⚙️ 詳細設定」）から変更できます。

### 音声認識エンジン

| 項目 | 説明 |
|------|------|
| **ローカル処理（推奨）** | PC 上で処理。無料・インターネット不要・ファイルサイズ制限なし |
| **OpenAI API** | OpenAI のサーバーで処理。高速だが有料（$0.006/分）・25MB 制限 |

### Whisper モデル（ローカル処理のみ）

| モデル | 精度 | 速度 | 用途 |
|--------|------|------|------|
| **turbo（推奨）** | ★★★★★ | ★★★★★ | 通常利用 |
| large-v3 | ★★★★★ | ★★ | 最高精度が必要な場合 |
| medium | ★★★★ | ★★★ | バランス型 |
| small | ★★★ | ★★★★ | 軽快に使いたい場合 |

### OpenAI API キー

OpenAI API モードを使う場合に入力します。  
入力したキーは自動的に保存され、次回以降は入力不要です。  
キーは `ハコ割り生成ツール_output/.hako_config.json` に保存されます。

### 認識言語

| 設定 | 説明 |
|------|------|
| **Japanese / 日本語（既定）** | 日本語に最適化。精度が最も高い |
| 自動検出 | 言語を自動判定。精度がやや下がる場合あり |
| その他 99 言語 | 英語・中国語・韓国語など、英語名 ABC 順で選択可 |

### タイミング設定

| 設定 | 既定値 | 説明 |
|------|--------|------|
| フレームレート (fps) | 30 | 字幕のタイムスタンプを合わせる fps |
| グローバルオフセット (秒) | 0.00 | 全タイムスタンプに加算するオフセット。音声と字幕がずれている場合に調整 |
| 後端パディング (秒) | 0.20 | 各ブロックの終了を何秒延ばすか |
| 最小表示時間 (秒) | 0.80 | 各字幕ブロックを最低何秒表示するか |
| 無音閾値 (ms) | 400 | この時間より短い無音区間は前ブロックを延ばして埋める |

**グローバルオフセットの使い方:**

```
音声が字幕より 0.3 秒遅れて聞こえる → グローバルオフセットを +0.30 に設定
音声が字幕より 0.3 秒早く聞こえる  → グローバルオフセットを -0.30 に設定
```

## 8. 活用例

### 例 1: 講演動画の字幕制作（最もよくある使い方）

**状況**: 60 分の企業セミナー動画（TS ファイル）に日本語字幕を付けたい。

**ファイル準備:**
```
seminar_20260601.ts     ← TS 形式の録画ファイル
seminar_hako.txt        ← 以下のようなハコ割りテキスト
```

`seminar_hako.txt` の例（空行区切り）:
```
本日はお忙しい中
お集まりいただき
ありがとうございます。

本日のテーマは
「DX 推進のための
AI 活用」です。

まず、現状の課題から
お話しします。
```

**操作:**
1. `seminar_20260601.ts` をアップロード
2. `seminar_hako.txt` をアップロード
3. 区切り: 「空行区切り」を選択
4. サイドバー: 言語 = `Japanese / 日本語`
5. 「▶ SRT を生成」をクリック
6. 処理時間: CPU で約 5〜10 分

**生成される SRT（例）:**
```
1
00:00:02,400 --> 00:00:05,800
本日はお忙しい中
お集まりいただき
ありがとうございます。

2
00:00:06,200 --> 00:00:10,100
本日のテーマは
「DX 推進のための
AI 活用」です。

3
00:00:10,600 --> 00:00:14,300
まず、現状の課題から
お話しします。
```

### 例 4: タイミング微調整

**状況**: 生成した SRT を再生してみると、すべての字幕が 0.5 秒ほど遅れていた。

**操作:**
- サイドバーの「グローバルオフセット」を `0.00` → `-0.50` に変更
- 「書き起こしキャッシュを使用」にチェックが入っていることを確認（再認識不要）
- 「▶ SRT を生成」を再実行
- Whisper の認識はスキップされ、タイミング計算のみ再実行（数秒で完了）

3. タイムラインにドラッグ → 映像クリップの上に配置
4. テキストサイズ・フォントをキャプションパネルで調整

## 9. 出力ファイルについて

| ファイル | 内容 |
|---------|------|

```
[ブロック番号]

```

## 10. トラブルシューティング

### アプリが起動しない

**症状**: 起動ショートカットをダブルクリックしてもブラウザが開かない。

確認事項:
1. Docker Desktop が起動しているか（タスクバー/メニューバーのアイコンを確認）
2. `http://localhost:8503` を手動でブラウザに入力してみる
3. コマンドウィンドウ/ターミナルのエラーメッセージを確認

---

### 初回起動でビルドが失敗する

**症状**: 「ビルドに失敗しました」と表示される。

確認事項:
- インターネット接続が正常か確認
- Docker Desktop が最新版か確認（古いバージョンは更新してください）
- セキュリティソフトが Docker の通信をブロックしていないか確認

---

### 音声認識の精度が低い

**考えられる原因と対処:**

| 原因 | 対処 |
|------|------|
| 言語設定が間違っている | サイドバーで正しい言語を選択 |
| BGM・ノイズが多い | 可能であれば音声をクリーニング |
| 方言・専門用語が多い | OpenAI API モードを試す（大規模モデル使用） |
| 小さいモデルを使用している | turbo または large-v3 に変更 |

---

### 字幕のタイミングがずれている

**ケース 1: 全体的にずれている**  
→ サイドバーの「グローバルオフセット」で調整してください（例: -0.30 など）

**ケース 2: 特定の箇所だけずれている**  
→ ハコ割りテキストと音声の内容が一致しているか確認してください

**ケース 3: 字幕が早く切れてしまう**  
→ 「後端パディング」の値を 0.2 → 0.5 程度に増やしてください

---

### ポート 8503 が使用中のエラー

**症状**: 「port is already allocated」というエラーが出る。

対処:
1. 「停止」ショートカットを実行してから再起動
2. または、既に起動している場合は `http://localhost:8503` にアクセスするだけで使えます

---

### Mac で「開発元を確認できない」と表示される

→ [インストール方法 > Mac](#mac) の Gatekeeper 対処を参照してください。

## 11. よくある質問（FAQ）

**Q: インターネットなしで使えますか？**  
A: 初回起動時のみインターネットが必要です（Whisper モデルのダウンロード）。  
2 回目以降は完全にオフラインで動作します。

**Q: 処理したファイルは外部に送られますか？**  
A: ローカル処理モードでは一切送信されません。完全にプライベートです。  
OpenAI API モードを使用した場合は、音声データが OpenAI に送信されます。

**Q: 処理時間の目安は？**  
A: CPU 環境での目安:

| 音声の長さ | turbo モデル |
|-----------|------------|
| 5 分 | 約 1〜2 分 |
| 30 分 | 約 5〜10 分 |

OpenAI API 使用時: 音声の約 1/5 〜 1/10 の時間

**Q: 複数のファイルを一度に処理できますか？**  
A: 現在は 1 ファイルずつの処理です。複数ある場合は順番に実行してください。

**Q: 生成した SRT の文字数に制限はありますか？**  
A: ありません。Whisper の認識結果に依存しますが、長い動画でも問題なく動作します。

**Q: ハコ割りテキストがない場合でも使えますか？**  
A: ハコ割りテキストは必須です。字幕の区切りを決める基準として使用しています。

**Q: Windows と Mac でクラウド共有（OneDrive, Dropbox）の出力フォルダは使えますか？**  
A: 使えます。出力先がデスクトップの `ハコ割り生成ツール_output` フォルダになっているため、  
そのフォルダをクラウド同期対象に設定すれば自動的に共有されます。

**Q: アンインストール方法は？**  
A: Windows: コントロールパネル → プログラムのアンインストール → ハコ割り生成ツール  
Mac: Applications から `ハコ割り生成ツール.app` を削除  
なお、Docker イメージを削除する場合は追加で以下を実行:
```
docker rmi hako-srt-app
```
